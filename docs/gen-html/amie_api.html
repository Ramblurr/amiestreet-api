Welcome!

The Amie Street Web Services API is a service based HTTP interface which on a basic level can return XML, JSON, or AMF-3 responses.

The endpoint for services are structured as follows:
http://amiestreet.com/api/{version}/{format}/{service}/{method}?param1=value1¶m2=value2¶mn=valuen

The current version of the API is 0.1.

{format} can be any one of xml, json, or amf.

Client Code

Having to write your own code to use an api sucks. You should be spending your time coming up with cool stuff, not slugging through a bunch of code just to try an idea out. That's why we generate clients automatically. Below are the clients in each language we already have available. We're always happy to add more, so just ask.

  • Python
  • PHP
  • Flex SWC
  • Javascript

Client Usage Examples

Javascript

The following example uses the ArtistApi service to get all albums by Mogwai ordered by popularity.

Code:
<h4>Top albums by Mogwai on Amie Street</h4>
<div id="topAlbumsResult"><!-- Albums go here --></div>
<!-- Include the Amie Street Javascript Client -->
<script type="text/javascript" src="http://amiestreet.com/static/js/amie-api-client.js"></script>
<script type="text/javascript">
amiestreet.ArtistApi_topAlbums('mogwai', function(albums){
$.each(albums, function(){
$('$topAlbumsResult').append('<a style="float: left; height: 120px; width: 120px;" href="'+this.url+'"><img src="'+this.image.medium+'" alt="'+this.title+' by '+this.artist.name+' on Amie Street" /></a>');
});
});
</script>

LibraryApi

Whenever someone purchases something on Amie Street, it is added to their library. The library api allows you to access various information about the library, i.e. how many songs are there, what artists has this user purchased from, etc. These methods require authentication in some cases as users can specify their libraries to be private.

numSongs

Get the number of songs in the user's library. 

i32 numSongs(userIdentifierRef user)

songs

Get a paged list of all songs in a user's library that have not been banned. 

PagedSongListVO songs(userIdentifierRef user,
                      i32 page,
                      i32 limit)

artists

Get a list of all artists in a user's library. 

list<ArtistVO> artists(userIdentifierRef user)

albums

Get a list of all albums in a user's library. 

list<AlbumVO> albums(userIdentifierRef user)

genres

Get a list of all genres which appear in a user's library. 

list<GenreVO> genres(userIdentifierRef user)

banSong

Ban a single song from the library. 

bool banSong(userFingerprint fingerprint,
             string songId)

banSongList

Ban a list of songs from the library. 

bool banSongList(userFingerprint fingerprint,
                 string songIdList)

unbanSong

Unban a single song which makes a once banned song show up in the Library.songs list again. 

bool unbanSong(userFingerprint fingerprint,
               string songId)

unbanSongList

Unban a list of songs which makes a once banned songs show up in the Library.songs list again. 

bool unbanSongList(userFingerprint fingerprint,
                   string songIdList)

bannedSongs

Get a list of all songs banned by the user. 

list<SongVO> bannedSongs(userIdentifierRef user)

refreshLibrary

Used by players as a periodic updater to see if the user has purchased any new songs since the app's initial song list was fetched. 

list<SongVO> refreshLibrary(userIdentifierRef user,
                            string songIds)

requestOfflineToken

If you're writing an app. Do not store passwords. At all. Allows you (as the application) to request a permanent token to access this user's account for most actions provided they grant you access with their username and password. 

string requestOfflineToken(string username,
                           string password,
                           string appName)

useOfflineToken

If you have an offline token stored for a user and would like to use that to get a fingerprint to user for normal auth procedures. 

AuthUserVO useOfflineToken(string token,
                           string appName)

NewsletterApi

Provides access to the Amie Street newsletter archives. The most common application is to call NewsletterApi.latest(0,0) (which obviously gets the latest newsletter marked as sent) and then NewsletterApi.items to get everything that was sent out in the email.

get

Gets simple meta data for a single newsletter. @param string id The newsletter unique id.

latest

Gets a list of newsletters sorted by the date they were marked as sent. @param int limit Max number to fetch. @param int offset Paging support. 

list<NewsletterVO> latest(i32 limit,
                          i32 offset)

items

Get a list of NewsletterItemVO's which were included in this newsletter. @param string id The newsletter unique id. 

list<NewsletterItemVO> items(string id)
